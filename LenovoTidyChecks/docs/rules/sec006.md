# SEC006 - 正则 DoS 风险（Regex DoS）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC006` / `lenovo-sec006-regex-dos-risk` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC006` in AnalyzerRules.md](../../AnalyzerRules.md#sec006---regex-dos-risk) |

## 描述

C++ 标准库 `std::regex` 的引擎是回溯式实现，对类似 `(a+)+`、`(a|aa)*` 的灾难性回溯模式没有防御机制，且**没有任何超时参数**。在用户可控输入上使用时容易被构造出 ReDoS 攻击。

本规则会对每个 `std::regex` / `std::wregex` 的构造点发出警告，提示改用：

- `re2::RE2`（Google RE2，线性时间）
- 或在调用处加超时（用 `std::async` + future timeout 包一层）
- 或对输入做长度上限校验

## 触发示例

```cpp
#include <regex>
std::regex re(userPattern);   // ❌
std::regex re("(a+)+");        // ❌
```

## 合规示例

```cpp
#include <re2/re2.h>
re2::RE2 re("(a+)+");          // 线性时间，安全
```

## 已知限制

- 即使是 const 字符串字面量构造的 regex 也会触发——这是刻意设计，避免假阴性
- 可在确定安全的位置用 `// NOLINT(lenovo-sec006-regex-dos-risk)` 抑制
