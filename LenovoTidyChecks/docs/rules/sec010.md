# SEC010 - 竞态条件（Race Condition）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC010` / `lenovo-sec010-race-condition` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC010` in AnalyzerRules.md](../../AnalyzerRules.md#sec010---race-condition) |

## 描述

启发式：命名空间/全局作用域的可变变量（非 `const`、非 `constexpr`、非 `thread_local`、不是 `std::atomic` / `std::mutex` 系），是多线程程序里数据竞争的最常见温床。本规则在每个这样的全局上发出警告。

## 触发示例

```cpp
int g_counter = 0;       // ❌
static int s_state = 0;  // ❌
namespace { double cache = 0.0; }   // ❌
```

## 合规示例

```cpp
const int kMaxRetry = 3;          // 常量
std::atomic<int> g_counter{0};    // 原子
thread_local int t_state = 0;     // 线程局部
std::mutex g_mu;                  // 互斥量本身用于同步
```

## 已知限制

- 仅看变量"形态"，不分析调用图，不区分单线程程序里的全局
- 函数内 `static` 变量已被排除（C++ 标准保证初始化是线程安全的）
- 推荐配合 ThreadSanitizer 等动态工具一起用
