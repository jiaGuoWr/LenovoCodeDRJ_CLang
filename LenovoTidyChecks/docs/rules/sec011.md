# SEC011 - 不安全 IPC（Insecure IPC）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC011` / `lenovo-sec011-insecure-ipc` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC011` in AnalyzerRules.md](../../AnalyzerRules.md#sec011---insecure-ipc) |

## 描述

C# 检测的是 WCF 的不安全绑定，C++ 侧适配为现代 IPC 框架中明确"不安全"的入口：

- `grpc::InsecureChannelCredentials` / `grpc::InsecureServerCredentials`
- C 风格 `grpc_insecure_channel_create`
- 未来可扩展到 ZeroMQ `zmq_socket` 上无 CURVE 认证的绑定

## 触发示例

```cpp
auto channel = grpc::CreateChannel("svc:50051", grpc::InsecureChannelCredentials());  // ❌
```

## 合规示例

```cpp
auto channel = grpc::CreateChannel("svc:50051",
    grpc::SslCredentials(grpc::SslCredentialsOptions{}));   // ✅
```

## 已知限制

- 仅匹配函数名级别；如果项目内自封了 wrapper 但底层仍走 Insecure，需手动加进白名单
