# SEC004 - 不安全反序列化（Unsafe Deserialization）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC004` / `lenovo-sec004-unsafe-deserialization` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC004` in AnalyzerRules.md](../../AnalyzerRules.md#sec004---unsafe-deserialization) |

## 描述

C# 侧检测的是 `BinaryFormatter`，C++ 侧适配为常见的"无 schema 信任输入"反序列化入口：

- `boost::archive::binary_iarchive` / `text_iarchive` / `xml_iarchive`
- `cereal::BinaryInputArchive` / `JSONInputArchive` / `PortableBinaryInputArchive` / `XMLInputArchive`
- `msgpack::unpacker`

这些 API 都允许反序列化任意类型，攻击者可构造恶意流触发 RCE。

## 触发示例

```cpp
boost::archive::binary_iarchive ar(stream);   // ❌
cereal::BinaryInputArchive ar(stream);        // ❌
```

## 合规示例

- 使用强 schema 的格式：Protobuf、FlatBuffers、Cap'n Proto
- 或先用 JSON Schema / 自定义校验器验证数据完整性，再反序列化
- 仍用 boost/cereal 的，**只在受信任的内部信道**（管道、本进程内）

## 配置（v0.3 计划）

未来支持白名单：

```yaml
CheckOptions:
  lenovo-sec004-unsafe-deserialization.AllowedTypes: 'MyTrustedArchive,InternalIpcReader'
```
