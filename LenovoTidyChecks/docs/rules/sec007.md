# SEC007 - 资源泄漏（Resource Leak）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC007` / `lenovo-sec007-resource-leak` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC007` in AnalyzerRules.md](../../AnalyzerRules.md#sec007---resource-leak) |

## 描述

检测两类高频资源泄漏模式：

1. C++ 裸 `new`：未被 `std::make_unique` / `std::make_shared` / `std::unique_ptr` / `std::shared_ptr` 直接接管
2. C 风格 `fopen` / `tmpfile` / `freopen`：返回 `FILE*` 没有用 RAII 包装

## 触发示例

```cpp
Widget* w  = new Widget();        // ❌
int* arr   = new int[100];         // ❌
std::FILE* f = std::fopen(...);    // ❌
```

## 合规示例

```cpp
auto w = std::make_unique<Widget>();        // ✅
auto v = std::vector<int>(100);              // ✅
std::ifstream in("a.txt");                   // ✅ RAII
```

## 已知限制

- 是结构性启发，不是路径敏感分析；某些被 unique_ptr 间接接管的写法仍可能误报
- 用 NOLINT 行内豁免遗留代码
