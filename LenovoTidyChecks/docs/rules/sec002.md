# SEC002 - 路径遍历风险（Path Traversal）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC002` / `lenovo-sec002-path-traversal` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC002` in AnalyzerRules.md](../../AnalyzerRules.md#sec002---path-traversal-risk) |

## 描述

启发式检测：变量名看起来像文件路径（`path` / `dir` / `file` / `location` / `uri` / `*name`）且通过字符串拼接（`operator+`）构造。这类写法常见于路径遍历漏洞。

## 触发示例

```cpp
std::string filePath = "/base/" + userInput;        // ❌
std::string dirPath  = std::string("/var/log/") + sub;  // ❌
```

## 合规示例

```cpp
#include <filesystem>
std::filesystem::path p = std::filesystem::weakly_canonical(
    std::filesystem::path("/base") / userInput);
if (p.string().rfind("/base", 0) != 0) throw std::runtime_error("invalid");
```

## 已知限制

- 仅按变量名 + 拼接形态匹配，无法判断 `userInput` 是否真的来自外部
- 不检测 `sprintf` / `std::format` 形式的拼接（建议未来加入）
