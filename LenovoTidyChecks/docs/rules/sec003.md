# SEC003 - SQL 注入风险（SQL Injection）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC003` / `lenovo-sec003-sql-injection` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC003` in AnalyzerRules.md](../../AnalyzerRules.md#sec003---sql-injection-risk) |

## 描述

匹配以 `SELECT` / `INSERT` / `UPDATE` / `DELETE` / `MERGE` / `EXEC[UTE]` 开头的字符串字面量参与到 `+` 拼接表达式中。

## 触发示例

```cpp
std::string sql = "SELECT * FROM Users WHERE Name = '" + userName + "'";  // ❌
std::string q   = "DELETE FROM " + tableName + " WHERE id = 1";           // ❌
```

## 合规示例

```cpp
const char* sql = "SELECT * FROM Users WHERE Name = ?";  // 占位符
stmt.bind(1, userName);                                  // 通过驱动绑定
```

## 已知限制

- 仅识别 `+` 拼接；`std::format`、`fmt::format`、`sprintf` 不在覆盖范围（v0.3 计划补）
- 静态字符串拼接（`"a" "b"`）不会被误报
