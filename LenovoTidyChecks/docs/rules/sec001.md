# SEC001 - 硬编码敏感信息（Hardcoded Sensitive Info）

| 属性 | 取值 |
|---|---|
| **ID** | `SEC001` / `lenovo-sec001-hardcoded-sensitive` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`SEC001` in AnalyzerRules.md](../../AnalyzerRules.md#sec001---sensitive-information-in-code) |

## 描述

检测变量声明中"变量名看起来像敏感信息"且"初始化值为较长的字符串字面量"的情况，避免把密码、API Key、Token 之类的敏感信息硬编码到源码里。

匹配的变量名包括（不区分大小写，可通过 `SuspiciousNamePattern` 选项覆盖）：

- `password` / `passwd` / `pwd`
- `secret`
- `token`
- `api[_-]?key` / `access[_-]?key`
- `credential` / `credentials`

允许的前后缀组合示例：`user_password`、`api_key_header`、`pwd`。

## 消息格式

```
variable 'NAME' may contain hardcoded sensitive information [lenovo-sec001-hardcoded-sensitive]
```

## 触发示例

```cpp
// ❌ 触发 SEC001
const char* password = "MySecretPassword123";
const char* apiKey   = "sk-1234567890abcdef";
std::string secret   = "s3cret-value";
```

## 合规示例

```cpp
// ✅ 从环境变量 / 配置中心读取
const char* password = std::getenv("DB_PASSWORD");

// ✅ 把名字作为配置键，而不是真正的密钥
const char* apiKeyConfigName = "ApiKey";

// ✅ 占位值（空字符串、环境变量名等）会被跳过
const char* password = "";
const char* password_env_var = "DB_PASSWORD";
```

## 配置选项

在 `.clang-tidy` 的 `CheckOptions` 中可调整：

```yaml
CheckOptions:
  lenovo-sec001-hardcoded-sensitive.SuspiciousNamePattern: '^.*(password|secret|token).*$'
  lenovo-sec001-hardcoded-sensitive.MinimumLiteralLength: 6
```

| 选项 | 默认值 | 说明 |
|---|---|---|
| `SuspiciousNamePattern` | 见描述 | 变量名匹配的正则（`llvm::Regex`，大小写不敏感）。可按需加入公司内部命名 |
| `MinimumLiteralLength` | `6` | 字面量最小长度，短字符串会被放行以减少误报 |

## 抑制方式

### 行内

```cpp
const char* password = "placeholder";  // NOLINT(lenovo-sec001-hardcoded-sensitive)
```

### 区块

```cpp
// NOLINTBEGIN(lenovo-sec001-hardcoded-sensitive)
const char* password = "legacy_value";
// NOLINTEND(lenovo-sec001-hardcoded-sensitive)
```

### `.clang-tidy` 排除

```yaml
Checks: 'lenovo-*,-lenovo-sec001-hardcoded-sensitive'
```

## 已知限制

- 仅分析 `const char*` / `std::string` 这类窄字符串，不处理 `wchar_t`、`char16_t` 等宽字符字面量
- 当前版本不做数据流分析；若密码来自函数返回值但直接传给了赋值，会漏报
- 对 C++ 原生字符串拼接（如 `"MySecret" "123"`）和 raw string（`R"(...)"`）已覆盖
