# DLL003 - 不安全的动态库加载（Unsafe DLL Load）

| 属性 | 取值 |
|---|---|
| **ID** | `DLL003` / `lenovo-dll003-unsafe-dll-load` |
| **类别** | Security |
| **严重级别** | Warning |
| **默认启用** | 是 |
| **对齐 C#** | [`DLL003` in AnalyzerRules.md](../../AnalyzerRules.md#dll003---unsafe-dll-signature) |

## 描述

C# 端 DLL003 校验的是 DLL 数字签名，构建时由其它工具完成。C++ 侧本规则**适配为更可静态检测的等价问题**：动态库加载（`LoadLibrary*` / `dlopen`）传入相对路径或短名时，加载器会按 `%PATH%` 顺序查找，攻击者可在受控目录放同名 DLL 实现劫持。

## 触发示例

```cpp
HMODULE m = LoadLibraryA("user32.dll");        // ❌ 短名
void*   h = dlopen("libfoo.so", RTLD_LAZY);    // ❌ 短名
HMODULE n = LoadLibraryA(userInput);           // ❌ 非字面量也保守警告
```

## 合规示例

```cpp
HMODULE m = LoadLibraryA("C:\\Windows\\System32\\user32.dll");   // ✅ 绝对路径
void*   h = dlopen("/usr/lib/libfoo.so", RTLD_LAZY);              // ✅
// 或更严格：用 LoadLibraryEx + LOAD_LIBRARY_SEARCH_SYSTEM32
```

## 已知限制

- 不验证签名，签名校验仍需构建后另行处理
- 当首参数不是字符串字面量时，保守地报警；用 NOLINT 抑制误报
