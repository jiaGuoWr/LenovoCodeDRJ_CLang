# ADR-0002：跳过 DLL002 与 SEC009 两条规则

- 状态：Accepted
- 日期：2026-04-27

## 背景

C# 侧 `AnalyzerRules.md` 共 17 条规则。`v0.2` 计划把所有规则移植到 C++。
其中两条**与 C# 语言特性强绑定**，无法在 C++ 找到等价物：

| 规则 | C# 含义 | C++ 等价物 |
|---|---|---|
| DLL002 | 检测 `[DllImport]` 缺少 `[DefaultDllImportSearchPaths]` | C++ 没有 P/Invoke 这一概念 |
| SEC009 | 检测不安全反射（`Type.GetType("...")` 等）| C++ 没有反射运行时 |

## 候选

- a) 强行找 C++ 类比硬做（DLL002 → 检测 `LoadLibrary` 短名；SEC009 → 检测 `typeid` + 字符串）
- b) 标 N/A，仅 ADR + AnalyzerRules.md 头部说明
- c) 完全跳过，C++ 这边声称 15 条覆盖

## 决策

选 **c**：

- 跳过 DLL002，因为它的 C++ 等价物**已经被 DLL003 覆盖**（`LoadLibrary` 短名加载）
- 跳过 SEC009，C++ 没有反射，硬做容易引入大量误报
- `AnalyzerRules.md` 头部声明这两条 N/A，`scripts/generate_rules_index.py` 把它们列为
  "informational only"，不算覆盖率缺口

## 影响

- 规则总数：C# 17 → C++ 15
- 业务团队跨语言对照清单更清晰，不用为不存在的语言能力做特殊处理
- 未来若 C++ 引入 `std::reflexpr` / `static_reflection` 提案，可重新评估 SEC009
